# Project 4 - Sweep Signal

一个基于 **Pygame** 实现的竖屏小球规则实验项目。  
在这个版本中，小球需要在不断摆动的圆环缺口中寻找逃逸机会；如果在限定时间内未成功穿过缺口，小球会被冻结在当前位置，并生成下一颗球继续挑战。

相较于前几个项目，本项目在保留“小球 + 圆环 + 倒计时 + 冻结累积”核心框架的基础上，进一步加入了：

- **动态摆动缺口**
- **暖色主题 UI**
- **碰撞驱动的长音乐推进机制**
- **轻量碰撞音效辅助反馈**

整体目标不是做一个传统游戏，而是做一个更偏向 **规则实验 / 视听反馈 / 过程展示** 的小球项目。

---

## 项目预览

本项目的核心过程如下：

1. 小球从中心区域附近随机生成
2. 小球在圆环内部不断运动与反弹
3. 圆环缺口会在一定范围内持续摆动
4. 小球需要在倒计时结束前穿过缺口逃出圆环
5. 若未成功逃出，则当前小球被冻结在场上
6. 新的小球继续生成并开始下一轮挑战
7. 场上冻结球会逐步增加，进一步改变后续球的运动路径与难度

---

## 核心特性

### 1. 动态缺口机制
与静态缺口不同，本项目中的圆环缺口会在一定角度范围内往返摆动。  
随着倒计时减少，缺口摆动速度会逐渐提升，使后半段挑战更紧张。

### 2. 倒计时冻结机制
每颗小球拥有固定倒计时。  
若在倒计时结束前未成功穿过缺口，则会被冻结为场上的障碍物，进入下一轮。

### 3. 冻结累积与路径干扰
被冻结的小球不会消失，而是持续保留在场内。  
这些冻结球会在后续回合中与新生成的小球发生碰撞，从而形成越来越复杂的运动路径。

### 4. 方案A音频设计
本项目采用了一种相对实验性的音频反馈方案：

- 小球碰撞时，会触发**轻量碰撞音效**
- 同时，碰撞还会推动一首长音乐继续播放一小段
- 当没有新的碰撞时，长音乐暂停
- 下一次碰撞时，长音乐从上一次停止的位置继续

这种设计不是传统游戏式音效方案，而更接近一种“碰撞驱动音乐进度”的实验表达。

### 5. 暖色主题视觉风格
本项目相较于此前版本，整体 UI 和圆环颜色进行了重新设计，采用更偏暖色的视觉包装，以增强视频录屏时的识别度和差异化。


## 运行环境

* Python 3.11（推荐）
* Pygame 2.6.1（或相近版本）

安装依赖：

```bash
pip install pygame
```

运行项目：

```bash
python main.py
```

## 操作方式

* 启动程序后，按 **Space** 开始
* 小球会自动运动，无需额外控制
* 项目更偏向“观察型规则实验”，而不是传统手动操作型游戏

---

## 音频文件说明

本项目使用以下音频资源：

### 长音乐文件

```bash
collision_bgm.ogg
```

用途：

* 碰撞时推动长音乐继续播放一小段
* 推荐使用 **OGG** 格式，以获得更稳定的 `pygame.mixer.music.play(start=...)` 支持

### 轻碰撞音效

```bash
ring_hit.wav
frozen_hit.wav
```

用途：

* 为碰撞提供即时反馈
* 音量应保持较低，避免压过长音乐机制本身

### 其他提示音

```bash
freeze.wav
success.wav
```

用途：

* 倒计时结束冻结时播放
* 成功穿过缺口时播放

> 注意：如果 `collision_bgm.ogg` 无法正常加载，请优先检查文件是否为真正可用的 OGG 文件，而不是仅仅修改了后缀名的其他格式音频。

## 参数可调整项

在 `main.py` 中，可以根据需要调节以下参数：

### 缺口相关

* `GAP_SIZE_DEG`：缺口角度大小
* `GAP_SWING_RANGE_DEG`：缺口摆动范围
* `GAP_SWING_SPEED_MIN_DEG`：摆动最小速度
* `GAP_SWING_SPEED_MAX_DEG`：摆动最大速度

### 球与倒计时

* `BALL_SPEED`：小球速度
* `COUNTDOWN_SECONDS`：单球倒计时时长

### 长音乐推进

* `MUSIC_SEGMENT_SECONDS`：每次碰撞推动长音乐播放的时长
* `MUSIC_TOTAL_LENGTH_SECONDS`：长音乐总长度

### 碰撞音效

* `ring_hit.wav` 与 `frozen_hit.wav` 的音量
* 音效冷却时间 `ring_hit_cooldown_ms` / `frozen_hit_cooldown_ms`

## 设计思路

这个项目并不追求复杂操作，而更重视以下几点：

* 简单规则下的连续变化
* 冻结累积带来的场景演化
* 圆环缺口动态变化带来的不确定性
* 视觉与音频反馈共同构成的实验性观感

可以把它理解为一个偏“规则展示”和“过程观察”的小球项目，而不是一个传统关卡型作品。

---

## 当前版本特点与局限

### 优点

* 规则清晰，演示效果直观
* 动态缺口提升了过程的不确定性
* 冻结累积让每一轮都在改变场景结构
* 音频方案具有一定实验性和新颖感

### 局限

* 长音乐推进机制在不同音乐素材下效果差异较大
* 该音频方案更偏实验表达，不一定比传统碰撞音效更稳定
* 项目当前仍以单文件结构为主，后续可继续模块化整理
